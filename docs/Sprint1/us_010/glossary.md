# Glossary — AISafe Flight Control System

This glossary defines all domain concepts present in the domain model diagram. Every class in the diagram has exactly one entry here, and every entry corresponds to exactly one class in the diagram. Each entry identifies whether the concept is an Entity or a Value Object.

| **Term** | **Type** | **Description** |
|:---|:---|:---|
| **AerodynamicCoefficients** | Value Object | Groups wing area (m²), drag coefficient (Cd), and lift coefficient (Cl) of an aircraft model. These three parameters are always used together in the lift and drag physics formulas: L = Cl × A × ρv²/2 and D = Cd × A × ρv²/2 (section 3.3). Constitutive part of AircraftModel. |
| **AirControlArea** | Entity (root) | A geographic area of airspace used for air traffic control. Identified by a unique area code and defined by valid rectangular geographic boundaries. US050: "The area code must be unique in the system. Geographic boundaries must be valid." Client clarification: "For sake of simplicity, in the project it may be a rectangle." Boundaries cannot overlap; airport coordinates must fall within the area's boundary. |
| **Aircraft** | Entity (root) | An aircraft of a specific model operated by an air transport company. Identified by a unique registration number and country of registration. Has a cabin configuration and number of flight crew members. Section 3.2, US070. |
| **AircraftModel** | Entity (root) | A motorised commercial aircraft model. Characterised by a model ID, manufacturer, type, motorisation, weights, performance parameters, and aerodynamic coefficients. Name and manufacturer combination must be unique. All engine models certified for a given aircraft model must be of the same motorisation type. Section 3.2, US055. |
| **AircraftPerformance** | Value Object | Groups service ceiling, cruise speed, and maximum range of an aircraft model. Each value has its own unit defined at implementation time. These are cohesive performance parameters listed in section 3.2 and US055. Constitutive part of AircraftModel. |
| **AircraftType** | Value Object (enum) | The commercial purpose of an aircraft model. One of: passenger, cargo, mixed. Section 3.2. |
| **AircraftVariant** | Entity (internal) | A specific combination of an aircraft model and a certified engine model. Exists only within AircraftModel — no independent lifecycle. References EngineModel by ID only, respecting aggregate boundaries. US057: "An aircraft model might have several aircraft variants (combinations of model and engine configuration)." |
| **AircraftWeights** | Value Object | Groups empty weight, MTOW (Maximum Take-Off Weight), MZFW (Maximum Zero Fuel Weight), and maximum fuel capacity of an aircraft model. Invariant: MTOW > MZFW > emptyWeight, all values positive. Section 3.2. Constitutive part of AircraftModel. |
| **AirTransportCompany** | Entity (root) | A company that uses the system to register aircraft and flights. Has a unique company name, IATA designator (2 letters), and ICAO code (2–3 letters). US060. |
| **AreaCode** | Value Object | The unique identifier of an AirControlArea. Business rule: unique in the system. US050. Constitutive part of AirControlArea. |
| **ATCCollaborator** | Entity (internal) | A specialisation of Collaborator belonging to an AirTransportCompany. Can register aircraft, create flight routes, and manage pilots. Section 3.1.3. |
| **CabinConfiguration** | Value Object | The seating arrangement of an aircraft, composed of one or more SeatClass entries. Business rule: total seats cannot exceed the model's capacity. Section 3.2, US070. Constitutive part of Aircraft. |
| **CharterSchedule** | Value Object | Specialisation of DepartureSchedule for charter flights. Contains a specific departure date and departure time. Client: "Charter flights will have only a single instance." Section 3.2: "actual date for a charter." Constitutive part of DepartureSchedule hierarchy. |
| **Collaborator** | Entity (root, abstract) | A person with access to the system who works for a customer (AirTransportCompany or AirControlArea). Has a name, position, security clearance, and periodic skills assessment. Linked to a SystemUser for authentication. Abstract — never instantiated directly. Specialised into ATCCollaborator, FlightControlOperator, WeatherPerson. Section 3.1. |
| **CompanyName** | Value Object | The name of an AirTransportCompany. Business rule: unique. US060. Constitutive part of AirTransportCompany. |
| **Coordinates_ACA** | Value Object | The rectangular geographic boundary of an AirControlArea, defined by minLatitude, maxLatitude, minLongitude, and maxLongitude. Client clarification: "For sake of simplicity, in the project it may be a rectangle." Distinct from Coordinates_Apt and Coordinates_WD which represent single geographic points. Constitutive part of AirControlArea. |
| **Coordinates_Apt** | Value Object | The geographic location of an airport, defined by latitude and longitude (single point). Business rules: latitude between -90 and 90, longitude between -180 and 180. Same class in code as Coordinates_WD — duplicated in diagram for presentation. US052. Constitutive part of Airport. |
| **Coordinates_WD** | Value Object | The geographic point within an AirControlArea where a wind condition reading was taken, defined by latitude and longitude. Client: "Weather conditions are not the same everywhere inside an air control area." Same class in code as Coordinates_Apt — duplicated in diagram for presentation. Constitutive part of WindCondition. |
| **DepartureSchedule** | Value Object (abstract) | The departure schedule of a flight. Abstract — always instantiated as either CharterSchedule or RegularSchedule depending on the flight type. Has no attributes of its own — exists solely to allow Flight to have a single polymorphic composition so a flight always has exactly one schedule regardless of type. Section 3.2. Constitutive part of Flight. |
| **Elevation** | Value Object | The altitude of an airport in metres above sea level. Has a value and unit. Business rule: must be non-negative. US052: "elevation in meters above sea level." Constitutive part of Airport. |
| **EngineModel** | Entity (root) | An engine model that can be certified for use on aircraft models. Characterised by a name, manufacturer, motorisation type, power output, thrust (at two speeds), fuel consumption efficiency (TSFC), and fuel type. Name and manufacturer combination must be unique. US056. |
| **EngineName** | Value Object | The name of an EngineModel. Business rule: name and manufacturer combination must be unique. US056. Constitutive part of EngineModel. |
| **Flight** | Entity (root) | An instantiation of a FlightRoute on a specific schedule. Characterised by a unique flight designator, type (regular or charter), departure schedule, scheduled arrival time, assigned aircraft, and assigned pilot. Has one or more flight plans. Section 3.4: "a flight is an instantiation of a route." Section 3.2, US080. |
| **FlightControlOperator** | Entity (internal) | A specialisation of Collaborator belonging to an AirControlArea. Responsible for testing flight plans and simulating air control. Section 3.1.4. |
| **FlightDesignator** | Value Object | The unique identifier of a flight. Format: airline designator (2 letters) + numeric flight number (up to 4 digits) + optional one-letter operational suffix — xxn(n)(n)(n)(a). Business rule: unique. Section 3.2. Constitutive part of Flight. |
| **FlightPlan** | Entity (internal) | A document describing how a flight takes place, including fuel quantity. Created in draft status; weather data is added later; then validated or rejected. A flight may have multiple plans — if one is rejected, a new one is submitted. US080, US082, US085. Internal to Flight — no lifecycle outside it. |
| **FlightPlanStatus** | Value Object (enum) | The lifecycle state of a flight plan. One of: draft, validated. US080: "Flight plan status is set to draft until validated." |
| **FlightRoute** | Entity (root) | A route between two airports (origin and destination) owned by an AirTransportCompany. Has a unique route name and a deactivation date. No more flights can be created on a deactivated route. US073, US074. Client: "A route is owned by an ATC. Its ID includes the company ID (e.g. TP123)." |
| **FlightType** | Value Object (enum) | The type of a flight. One of: regular (recurring, with specified departure days and times per entry), charter (single occurrence, with a specific departure date and time). Section 3.2. |
| **FuelQuantity** | Value Object | The quantity of fuel carried in a flight plan. Has a value and unit. Business rule: must be strictly positive. US080. Constitutive part of FlightPlan. |
| **IATACode** | Value Object | An IATA code identifying an airport (3 letters) or airline (2 letters). Business rule: unique worldwide. Same class in code — duplicated in diagram for Airport and AirTransportCompany. US052, US060. Constitutive part of Airport and AirTransportCompany. |
| **ICAOCode** | Value Object | An ICAO code identifying an airport (4 letters) or company (2–3 letters). Business rule: unique worldwide. Same class in code — duplicated in diagram for Airport and AirTransportCompany. US052, US060. Constitutive part of Airport and AirTransportCompany. |
| **Manufacturer** | Entity (root) | A company that manufactures aircraft models and/or engine models. A single manufacturer can cover both roles. Section 3.1.2: "only basic information (name and country?) needs to be stored." Client: "Yes, it can [be both aircraft and engine manufacturer]." |
| **ModelID** | Value Object | The identifier of an AircraftModel. Business rule: name and manufacturer combination must be unique. US055. Constitutive part of AircraftModel. |
| **MotorizationType** | Value Object (enum) | The type of motorisation of an engine model. One of: turboprop, turbofan, turbojet, ramjet, electricPropeller. Section 3.2. US055: "all engines are of the same type" within an aircraft model variant. |
| **OperationalStatus** | Value Object (enum) | The operational state of an aircraft. One of: active, decommissioned. US071: "It's not possible to create flights for a retired aircraft." |
| **Pilot** | Entity (root) | A collaborator of an AirTransportCompany certified to pilot one or more aircraft models. Responsible for registering and validating flight plans. Cannot be deactivated with assigned flights. Separated as its own aggregate because it has an independent lifecycle (US075–077), a specific invariant (US077), and is directly referenced by Flight. Inherits common collaborator attributes from Collaborator. US075, US077. |
| **Power** | Value Object | The power output of an engine model. Has a value and unit. Business rule: must be positive. US056: "power" listed as important engine information. Constitutive part of EngineModel. |
| **RegularSchedule** | Value Object | Specialisation of DepartureSchedule for regular flights. Contains 1..* ScheduleEntry instances — one per departure day, each with its own departure time. Client: "Monday 12:00; Tuesday 12:30; Thursday 11:30." Constitutive part of DepartureSchedule hierarchy. |
| **RegistrationNumber** | Value Object | The unique registration of an aircraft, including registration number and country of registration. Business rule: unique worldwide. US070: "An aircraft is identified by the aircraft registration number, that is unique. Also, an aircraft is registered in a country (it may not be the company's home country)." Constitutive part of Aircraft. |
| **RouteName** | Value Object | The name of a FlightRoute. Format: 2 letters (company initials) + up to 4 digits (e.g. TP123). Business rule: unique. US073. Client: "A route is owned by an ATC. Its ID includes the company ID." Constitutive part of FlightRoute. |
| **SafetyThreshold** | Value Object | The safety threshold parameter of a simulation. Has a value and unit. Business rule: must be positive. US100: "safety thresholds" listed as a simulation parameter. Constitutive part of Simulation. |
| **ScheduleEntry** | Value Object | One entry in a RegularSchedule, representing a single departure day and time. Contains a day of the week and a departure time. Client: "Monday 12:00; Tuesday 12:30; Thursday 11:30" — each pair is one ScheduleEntry. Constitutive part of RegularSchedule. |
| **SeatClass** | Value Object | One class of seating within a CabinConfiguration. Has a class name and a number of seats. Constitutive part of CabinConfiguration. Section 3.2: "cabin configuration (number of seats in each class)." |
| **SecurityClearance** | Value Object | The security clearance of a collaborator. Contains the expiry date. Business rule: automatically expires at a given date. Section 3.1.1: "An AISafe user needs to have an active security clearance that automatically expires at a given date." Constitutive part of Collaborator. |
| **Simulation** | Entity (root) | A simulation of flights in a given AirControlArea. Defined by time range, safety thresholds, included flights, and optional weather data. Produces a simulation report. Client: "You have to send information to run the simulation and you receive feedback/results from the simulation module. This information must be in the system." US100. |
| **SimulationReport** | Value Object | The outcomes of a simulation. Contains total number of flights and an overall validation result (passed/failed). Final immutable document received from the C simulation module. US109, US111. Constitutive part of Simulation. |
| **SimulationTimeRange** | Value Object | The time range of a simulation. Contains start and end date/time. Business rule: start must be before end. US100: "time range" listed as a simulation parameter. Constitutive part of Simulation. |
| **SkillsAssessment** | Value Object | The periodic skills assessment of a collaborator. Contains the assessment date. Business rule: must be within the last 5 years. Section 3.1.1: "They also need to have periodic (per regulations 5 years) skills assessment." Constitutive part of Collaborator. |
| **SystemUser** | Entity (framework boundary) | A user of the system managed by the EAPLI framework. Stores email (unique valid email), name, and phone number. Section 3.1.1: "A user is identified by a unique valid email. A user also has a name and phone number." The domain model represents collaborators up to this boundary — internal details managed by the framework. |
| **Thrust** | Value Object | The thrust produced by an engine at a given speed reference (static or cruise). Has a value, unit, and speed reference. Section 3.3: "Thrust is a parameter supplied by the manufacturer, usually at sea level and at two speeds: stopped and at cruise or maximum speed." An EngineModel always has exactly two Thrust instances. Constitutive part of EngineModel. |
| **Thrust Specific Fuel Consumption (TSFC)** | Value Object | The fuel efficiency of an engine design with respect to thrust output. Describes fuel consumption per unit of thrust. Has a value and unit. Section 3.3: "fuel consumption (grams/second) per unit of thrust (kN). But it can be expressed in other units." Constitutive part of EngineModel. |
| **ValidationResult** | Value Object (enum) | The overall outcome of a simulation. One of: passed, failed. US109, US111: "The report should indicate whether the scheduled flights plan passed or failed validation." |
| **WeatherData** | Entity (root) | Meteorological data registered for a specific AirControlArea at a given date and time. May come from multiple external providers. Contains localised wind condition readings because weather varies within an area. US041, US042. Client: "Weather conditions are not the same everywhere inside an air control area." |
| **WeatherPerson** | Entity (internal) | A specialisation of Collaborator belonging to an AirControlArea. Responsible for registering and importing weather data. US041, US042. |
| **WindCondition** | Value Object | A wind reading at a specific geographic point within an AirControlArea. Has a direction angle (degrees relative to North), speed (m/s), and a Coordinates_WD identifying where within the area the reading was taken. Section 3.2. Client: "Weather conditions are not the same everywhere inside an air control area." Constitutive part of WeatherData. |
